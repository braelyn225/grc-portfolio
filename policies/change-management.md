# Change Management Policy

## Purpose
Ensure changes to systems, applications, and infrastructure are controlled to minimize risk.

## Scope
Applies to all production systems, configurations, and code changes.

## Roles & Responsibilities
- **Change Owner:** Requests and documents changes
- **Approver:** Reviews and approves changes
- **Implementer:** Executes changes and validates outcomes

## Policy Statements
1. All changes require a documented request and approval.
2. Emergency changes must be documented within 24 hours.
3. Changes must include rollback plans.
4. Testing is required prior to production deployment.
5. Change records must include evidence of implementation and verification.

## Change Types
- **Standard:** Pre-approved, low-risk
- **Normal:** Requires review and approval
- **Emergency:** Urgent changes with expedited approval

## Review Cadence
Reviewed annually.
