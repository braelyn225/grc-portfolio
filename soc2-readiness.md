# SOC 2 Readiness Checklist

## Scope & System Description
- [ ] Define system boundaries and services
- [ ] Identify in-scope applications, infrastructure, and data
- [ ] Document data flows and third-party dependencies
- [ ] Draft/refresh system description

## Trust Services Criteria (TSC)
### Security (Common Criteria)
- [ ] CC1: Control environment
- [ ] CC2: Communication and information
- [ ] CC3: Risk assessment
- [ ] CC4: Monitoring activities
- [ ] CC5: Control activities
- [ ] CC6: Logical and physical access controls
- [ ] CC7: System operations
- [ ] CC8: Change management
- [ ] CC9: Risk mitigation

### Availability (If in scope)
- [ ] A1: Availability commitments
- [ ] A2: Availability monitoring and incident response

### Confidentiality (If in scope)
- [ ] C1: Confidentiality commitments
- [ ] C2: Confidential information handling

### Processing Integrity (If in scope)
- [ ] PI1: Processing integrity commitments
- [ ] PI2: Data processing controls

### Privacy (If in scope)
- [ ] P1: Privacy notice and consent
- [ ] P2: Collection and retention
- [ ] P3: Access and correction
- [ ] P4: Monitoring and enforcement

## Control Evidence
- [ ] Policies and procedures approved
- [ ] Access reviews completed
- [ ] Change management records
- [ ] Logging and monitoring evidence
- [ ] Incident response exercises
- [ ] Vendor risk assessments
